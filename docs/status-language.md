# Status Language

## Purpose

This repository uses explicit status labels so public readers can distinguish a published repository from a released program, report, model, dataset, Space, school, NEURONA, or deployment.

## Approved Labels

| Status | Meaning |
| --- | --- |
| `planned` | Named and review-routed, but not public as an artifact. |
| `scaffolded` | Public structure, templates, standards, or synthetic examples exist, but no release claim is made. |
| `published` | The public repository or document is reachable and intended for public review. |
| `released` | A reviewed artifact is public with approval evidence, limitations, and release notes. |
| `paused` | Visible work is intentionally not advancing or should not be relied on as active. |
| `deprecated` | Visible work remains for archive or transition context only. |
| `private/not-public` | Work exists or may exist outside public surfaces and must not be linked or summarized as public proof. |

## Rules

- `published` does not mean `released`.
- `scaffolded` does not mean live, operational, deployed, staffed, funded, or report-backed.
- Foundation programs, schools, NEURONAs, reports, datasets, models, Spaces, services, and deployments remain `planned` unless reviewed public evidence exists.
- Synthetic examples must remain labeled synthetic.
- Private, sealed, donor-sensitive, student-sensitive, volunteer-sensitive, customer-sensitive, security-sensitive, and unreleased material remains `private/not-public`.
