# Organization Standard

## Purpose

The 218-network GitHub organization is the 218 Network Foundation's institutional public repository surface. It is not Alexandra Caussade's personal profile, not the K-ussade technical brand, not a company namespace, and not a sealed development workspace.

## Authority Model

| Surface | Role | Voice |
| --- | --- | --- |
| Alexandra Caussade | Person, legal name, Founder identity, and approved public face. | Person/founder context. |
| K-ussade | Alexandra's public technical brand/account surface across GitHub, Hugging Face, and public proof navigation. | Technical proof and release-readiness discipline. |
| 218 Network Foundation | Civic institution. | Foundation civic voice, public trust, governance transparency, donor expectations. |
| 218.network | Canonical Foundation public website. | Foundation public web voice. |
| 218-network GitHub org | Institutional public repo surface. | Public documentation, governance, reports, roadmap, and program scaffolds. |
| YOSO-YAi LLC | Separate company voice and engineering partner where applicable. | Company obligations, engineering source-of-truth, sealed IP discipline. |
| YOSOR | Product of YOSO-YAi LLC. | Product surface, not Foundation identity. |

Caussade is not the brand name. K-ussade is the public technical brand/account surface.

## Repository Standard

Each public institutional repo must include:

- README with purpose, status, public data, and exclusions;
- AGENTS rules for AI-assisted changes;
- CONTRIBUTING and SECURITY rules;
- status language and boundary documentation;
- review gates;
- GitHub-renderable Mermaid graphs with purpose, interpretation notes, boundary notes, and follow-up actions;
- a validation script that checks required files and graph sections.

## Public Posture

The organization may publish public-safe scaffolds, standards, templates, synthetic examples, and approved reports. It must not become a storage surface for donor records, student records, volunteer private data, private Foundation operations, sealed YOSO-YAi LLC IP, customer data, private training corpora, secrets, or sensitive NEURONA operational details.
