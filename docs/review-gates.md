# Review Gates

## Purpose

Review gates prevent public scaffolding from being mistaken for live civic operations, released artifacts, or approved legal/financial records.

## Required Gates

| Gate | Required Before |
| --- | --- |
| Entity voice review | Any Foundation public copy, cross-link to K-ussade, or mention of YOSO-YAi LLC or YOSOR. |
| Status review | Any change from `planned` to `scaffolded`, `published`, `released`, `paused`, `deprecated`, or `private/not-public`. |
| Governance review | Any policy, board, conflict-of-interest, legal, or authority document presented as real. |
| Report review | Any transparency report, financial summary, donation disclosure, impact note, or public report presented as real. |
| Program review | Any cleanup, NEURONA mesh, or school status beyond public-safe scaffolding. |
| Privacy review | Any material that could involve donors, students, volunteers, customers, partners, locations, telemetry, or private operations. |
| Release review | Any model, dataset, Space, report, deployment, school, NEURONA, public service, or artifact marked `released`. |

## Refusal Rules

Refuse public publication when a change would expose private data, imply live operations without evidence, collapse Foundation and company voices, treat K-ussade as the Foundation institution, or treat Codex output as architectural authority.
